2.3 STATISTICAL ALGORITHMS
---------------------------

A) STANDARD DEVIATION CALCULATION
   Location: AnomalyDetectionStrategy.java (Line 85-95)
   Purpose: Detect price anomalies using statistical methods
   
   Algorithm:
   1. Calculate mean: average(values)
   2. Calculate variance: average((value - mean)²)
   3. Calculate std dev: sqrt(variance)
   
   Mathematical Formula:
   σ = √(Σ(xi - μ)² / n)
   
   Implementation:
   private double calculateStdDev(List<Payment> payments, double mean) {
       double variance = payments.stream()
           .mapToDouble(p -> Math.pow(val - mean, 2))
           .average()
           .orElse(0.0);
       return Math.sqrt(variance);
   }
   
   Complexity: O(n) - single pass through data
   Application: 3-sigma rule for anomaly detection (99.7% confidence)

B) AVERAGE/MEAN CALCULATION

2.4 DATA PROCESSING ALGORITHMS
-------------------------------

A) CHUNKING ALGORITHM (Divide and Conquer)
   Location: DatasetService.java
   Purpose: Process large files in manageable chunks
   
   Implementation:
   - CHUNK_SIZE = 5000 records
   - Break CSV into chunks
   - Process each chunk independently
   - Batch save to database
   
   Pseudocode:
   while (hasMoreRecords) {
       chunk = readNextChunk(CHUNK_SIZE)
       validRecords = filterValid(chunk)
       batchSave(validRecords)
   }
   
   Advantages:
   • Memory-efficient: O(CHUNK_SIZE) memory instead of O(n)
   • Enables streaming for files larger than RAM
   • Better cache locality
   • Predictable memory footprint
   
   Complexity: O(n) time, O(CHUNK_SIZE) space

B) DEDUPLICATION ALGORITHM
   Location: DatasetService.java (processCustomers, processOrders, etc.)
   Purpose: Remove duplicate records in real-time
   
   Algorithm:
   Set<String> seenIds = new HashSet<>();
   for each record in chunk:
       if not seenIds.contains(record.id):
           seenIds.add(record.id)
           validRecords.add(record)
       else:
           duplicatesRemoved++
   
   Complexity Analysis:
   • HashSet insert: O(1)
   • HashSet contains: O(1)
   • Total: O(n) for n records
   
   Comparison with naive approach:
   • Naive (nested loops): O(n²)
   • Optimized (HashSet): O(n)
   • Speed improvement: 99,000x for 100K records



C) FILTERING ALGORITHM
   Location: Stream-based filtering throughout strategies
   Purpose: Extract records meeting specific criteria
   
   Examples:
   • Anomaly detection: filter(p -> p.getValue() > threshold)
   • Delay analysis: filter(d -> d.getDelayHours() > 168)
   • Customer segmentation: filter by order count
   
   Complexity: O(n) linear scan
   Optimization: Predicate short-circuit evaluation

D) MAPPING/TRANSFORMATION ALGORITHM
   Location: Stream.map() operations
   Purpose: Transform data from one format to another
   
   Examples:
   • Entity → DTO conversion
   • Database results → JSON response
   • Calculation of derived fields
   
   Complexity: O(n)

E) LIMITING ALGORITHM (Top-N)
   Location: Stream.limit(N) operations
   Purpose: Get first N results
   
   Example:
   .filter(condition)
   .limit(20)
   .collect(Collectors.toList());
   
   Complexity: O(n) worst case, but early termination on limit
   Use: Prevents overwhelming frontend with too much data

3.3 OPTIMIZATION ACHIEVEMENTS
------------------------------

A) DEDUPLICATION OPTIMIZATION

BEFORE (Naive Approach):
for each record1 in dataset:
    for each record2 in dataset:
        if record1.id == record2.id:
            mark as duplicate

Complexity: O(n²)
For 100,000 records: 10,000,000,000 comparisons

AFTER (HashSet Approach):
Set<String> seen = new HashSet<>();
for each record in dataset:
    if seen.contains(record.id):
        mark as duplicate
    else:
        seen.add(record.id)

Complexity: O(n)
For 100,000 records: 100,000 operations

IMPROVEMENT: 99,000x faster (10B → 100K operations)



B) ANALYTICS OPTIMIZATION

BEFORE (N+1 Problem):
for each customer:
    orders = getOrdersForCustomer(customer.id)  // N database queries
    count += orders.size()

Complexity: O(n) database queries
Time: n × query_latency

AFTER (Aggregated Query):
results = repository.countOrdersByCustomer()  // Single query with GROUP BY

Complexity: O(1) database queries
Time: 1 × query_latency

IMPROVEMENT: From N queries to 1 query

C) MEMORY OPTIMIZATION

BEFORE (Load All Data):
List<Customer> allCustomers = loadAllFromFile(filePath)
// Memory: O(n) where n = total file size

AFTER (Chunk Processing):
while (hasChunks):
    chunk = readChunk(CHUNK_SIZE)
    process(chunk)
// Memory: O(CHUNK_SIZE) - constant memory

IMPROVEMENT: Can process files larger than available RAM

7.2 STATISTICAL ALGORITHM
---------------------------
File: AnomalyDetectionStrategy.java
Lines: 85-95

```java
private double calculateStdDev(List<Payment> payments, double mean) {
    double variance = payments.stream()
        .mapToDouble(p -> {
            double val = p.getPaymentValue() != null ? p.getPaymentValue() : 0.0;
            return Math.pow(val - mean, 2);
        })
        .average()
        .orElse(0.0);
    
    return Math.sqrt(variance);
}
```

• Mathematical algorithm (standard deviation)
• Stream processing
• Aggregation algorithm (average)
