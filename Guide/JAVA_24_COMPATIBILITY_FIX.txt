When running tests with Java 24, the following errors occurred:

1. **Mockito Mocking Errors**:

   - Root cause: Byte Buddy doesn't officially support Java 24 (only Java 23)
   - Affected: All tests using @Mock or @MockBean annotations

2. **Unit Test Assertion Failure**:
   - Test: DatasetServiceTest.testGetIngestionStats
   - Error: expected: <1000> but was: <null>
   - Root cause: Test was checking wrong key names in the stats map

   - Errors: 23 (Mockito/Byte Buddy compatibility)

================================================================================
SOLUTIONS APPLIED
================================================================================

### Fix 1: Enable Byte Buddy Experimental Mode for Java 24

Modified: pom.xml

Added maven-surefire-plugin configuration with experimental flag:

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>3.2.5</version>
    <configuration>
        <argLine>
            -Dnet.bytebuddy.experimental=true
            --add-opens java.base/java.lang=ALL-UNNAMED
            --add-opens java.base/java.util=ALL-UNNAMED
        </argLine>
    </configuration>
</plugin>
```

**Why this works**:
- `-Dnet.bytebuddy.experimental=true` tells Byte Buddy to enable experimental
  support for Java versions beyond officially supported ones
- `--add-opens` flags allow Mockito to access internal JDK classes needed
  for byte code manipulation

### Fix 2: Upgrade Byte Buddy to Latest Version

Modified: pom.xml

Added explicit Byte Buddy dependencies with version 1.15.11:

```xml
<!-- Byte Buddy for Java 24 compatibility -->
<dependency>
    <groupId>net.bytebuddy</groupId>
    <artifactId>byte-buddy</artifactId>
    <version>1.15.11</version>
    <scope>test</scope>
</dependency>

<dependency>
    <groupId>net.bytebuddy</groupId>
    <artifactId>byte-buddy-agent</artifactId>
    <version>1.15.11</version>
    <scope>test</scope>
</dependency>
```

**Why this works**:
- Version 1.15.11 is the latest Byte Buddy release (as of June 2026)
- Has better experimental support for newer Java versions
- Overrides the older Byte Buddy version bundled with Mockito

### Fix 3: Correct Unit Test Assertions

Modified: backend/src/test/java/com/transitflow/service/DatasetServiceTest.java

Changed assertion keys to match actual DatasetService.getIngestionStats() output:

**BEFORE (Wrong keys)**:
```java
assertEquals(1000L, stats.get("customers"));
assertEquals(1500L, stats.get("orders"));
assertEquals(2500L, stats.get("order_items"));
// etc.
```

**AFTER (Correct keys)**:
```java
assertEquals(1000L, stats.get("total_customers"));
assertEquals(1500L, stats.get("total_orders"));
assertEquals(2500L, stats.get("total_order_items"));
assertEquals(2000L, stats.get("total_payments"));
assertEquals(500L, stats.get("total_products"));
assertEquals(300L, stats.get("total_sellers"));
assertEquals(1400L, stats.get("total_deliveries"));
assertEquals(10L, stats.get("faulty_records"));
```

Also added missing mock for faultyRecordRepository.count():
```java
when(faultyRecordRepository.count()).thenReturn(10L);
```

**Why this was needed**:
- The DatasetService returns keys with "total_" prefix
- The test was checking for keys without the prefix
- This caused null pointer assertions to fail

================================================================================
TECHNICAL DETAILS
================================================================================

**Java Version Issues**:
- You are using Java 24 (java.version=24.0.1)
- Mockito uses Byte Buddy for bytecode manipulation
- Byte Buddy officially supports up to Java 23 (class file version 67)
- Java 24 uses class file version 68

**Byte Buddy Experimental Mode**:
When `-Dnet.bytebuddy.experimental=true` is set:
1. Byte Buddy bypasses its version check
2. Attempts to process Java 24 class files anyway
3. Uses best-effort approach for unsupported versions
4. Usually works fine for minor version bumps

**Module System Considerations**:
Java 9+ module system restricts reflective access. The --add-opens flags:
- Allow Mockito to access java.lang and java.util internals
- Necessary for creating dynamic proxies and mocks
- Required for @Mock, @MockBean, and Mockito.mock() to work
